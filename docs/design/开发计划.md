### 关键路径
`M0 → M1 → M4 → M5`(M2、M3 并行喂入 M4)。M6/M7/M8 在 M4 之后可部分并行。

### 风险闸门(M1)
function calling 压测需验证:是否按 schema 返回、是否漏调用、参数是否正确、多轮是否退化。
- **通过** → 继续按 tool-calling 范式铺开。
- **不通过** → 启用兜底:更强约束的 tool prompt / ReAct 文本协议降级 / 关键步骤显式引导。先解决再进 M4。

### 一期"完成"定义(DoD)
一个跑在 DeepSeek 上的 CLI coding agent,能够:
- 用自然语言驱动:探索仓库(grep/symbol/repo-map)、读改文件(精确替换+重试)、跑 shell 命令(带权限确认与输出截断);
- 一轮并行多工具;
- 维护会话上下文(压缩)并可中断恢复;
- 加载项目规则、Skill、MCP 工具;
- 支持 `/effort` 调节用力程度;
- 推理模型 token 正确处理;
- CI 绿、有单测+集成测试。

### 测试策略(贯穿各里程碑)
- 每个工具/模块配单测(pytest);Executor、edit 重试、输出截断、压缩为重点。
- M4 起加 agent loop 集成测试(可用 mock provider 或录制回放)。
- M9 做端到端场景测试(在示例仓库上完成"读—改—验证"闭环)。
